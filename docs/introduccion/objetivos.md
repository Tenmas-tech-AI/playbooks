# Objetivos del Programa

## Meta principal

Convertir a Tenmás en una empresa **AI-Native**: donde el uso de IA en el desarrollo de software no es opcional ni experimental, sino el **estándar de trabajo** de todos los ingenieros.

## ¿Qué significa ser AI-Native?

No significa generar código al azar con ChatGPT. Significa:

- **Flujo integrado**: IA en el editor, en el code review, en los tests y en el monitoreo
- **Estándares definidos**: prompts probados, quality gates claros, métricas visibles
- **Mejora continua**: cada semana medimos, ajustamos y documentamos aprendizajes

## Objetivos por semana

### Semana 1 — Stack Base
- Configurar **Cursor AI** como editor estándar del equipo
- Integrar **CodeRabbit** en todos los repositorios de Tenmás
- Implementar **PromptLayer** para observabilidad de prompts
- Automatizar workflows con **Make.com**
- Cubrir features críticos con **Playwright** (testing E2E)

### Semana 2 — Madurez y Estándares
- Construir pipelines de IA con **LangChain**
- Crear dashboards de productividad en **Grafana**
- Publicar la documentación con **Docusaurus** (este sitio)
- Definir y automatizar **quality gates** para código generado con IA
- Entregar los **playbooks** reutilizables para el equipo

## KPIs del programa

| KPI | Objetivo |
|-----|----------|
| Velocidad de desarrollo (features/semana) | +40% vs baseline |
| Tiempo promedio de code review | −50% |
| Cobertura de tests | ≥ 80% |
| Prompts documentados y reutilizables | ≥ 20 |
| Documentación actualizada | 100% |

## Compromisos del equipo

Para que esto funcione, todos los ingenieros de Tenmás se comprometen a:

1. **Usar Cursor AI** como editor principal durante las 2 semanas
2. **No mergear PRs sin review de CodeRabbit** a partir del Día 1
3. **Registrar prompts efectivos** en PromptLayer cuando los descubran
4. **Actualizar este playbook** cuando encuentren algo que falta documentar
